# 2026-06-15 Workforce-Status(WFS) 프로젝트 수정 명세서

금일 진행된 Workforce-Status 프로젝트의 주요 기능 개선 및 디버깅 요약입니다.

---

## 1. 프로젝트 숨기기(Hide) 및 복원 기능 구현
화면에서 불필요한 프로젝트를 숨김 처리하여 대시보드를 깔끔하게 유지할 수 있는 기능을 신설했습니다.

*   **프로젝트 뷰 (Project View)**:
    *   각 프로젝트 행의 우측에 위치한 삭제(`🗑️`) 버튼 바로 옆에 숨기기(`👁️‍🗨️`) 버튼을 추가했습니다.
*   **그룹별 뷰 (Group View)**:
    *   기존에 삭제 버튼이 없었던 그룹별 리스트의 각 프로젝트 헤더(`tr.sub-header`) 우측에도 삭제(`🗑️`) 및 숨기기(`👁️‍🗨️`) 버튼을 동일하게 연동하여 사용자 접근성을 강화했습니다.
*   **상태 영속성 및 필터링**:
    *   숨긴 프로젝트 목록을 `localStorage`(`wfs_hidden_project_ids`)에 저장하여 브라우저 새로고침 후에도 유지되도록 처리했습니다.
    *   `transformDataByGroup` (그룹별 통계 및 구조화) 및 `filteredData` (프로젝트 뷰 데이터 가공) 로직에서 숨겨진 프로젝트는 대시보드 집계와 배정 목록에서 완전히 제외되도록 필터링을 구축했습니다.

---

## 2. 숨김 프로젝트 관리 위젯 추가 (우상단 툴바)
숨겨진 프로젝트들을 관리하고 복원할 수 있는 전용 컨트롤러를 탑재했습니다.

*   **상단 툴바 위젯**:
    *   설정 버튼 좌측에 **`숨김 관리 (숨긴 프로젝트 수)`** 버튼을 추가했습니다.
*   **목록 조회 및 복원**:
    *   클릭 시 숨겨진 프로젝트 이름 목록이 드롭다운 팝업으로 나타나며, 개별 프로젝트 우측의 **`표시`** 버튼을 눌러 언제든지 보드로 즉시 복구(숨김 해제)할 수 있습니다.
    *   숨겨진 모든 프로젝트를 일괄 복구할 수 있는 **`전체 해제`** 기능을 추가했습니다.
*   **사용성 개선**:
    *   드롭다운 영역 바깥 클릭 시 팝업이 자동으로 닫히도록 바깥 클릭 감지(`handleClickOutside`) 이벤트 리스너를 연동했습니다.

## 3. 화면 크래시 에러 수정 (ReferenceError 디버깅)
*   **현상**: 최근 추가한 숨김 핸들러(`handleHideProject`) 내에서 `setConfirmConfig`를 참조하는 과정에서 `ReferenceError: Cannot access 'setConfirmConfig' before initialization` 에러가 발생하여 화면이 렌더링되지 않던 문제가 있었습니다.
*   **원인**: `setConfirmConfig` 상태가 1750행 부근에서 정의되어 있던 반면, 이를 참조하는 `handleHideProject`가 그보다 윗줄(1200행 부근)에 선언되어 Temporal Dead Zone(TDZ) 참조 오류가 발생했습니다.
*   **해결**: `handleHideProject` 콜백 함수 선언부를 `confirmConfig` 상태 선언부의 하단(1750행 이하)으로 이동시켜 의존 관계 순서를 바로잡아 크래시 현상을 완벽히 해결했습니다.

## 4. 화면 진입 및 스크롤 시 렌더링 먹통 현상 (Rendering Freeze) 개선
대형 그리드 데이터를 다루는 배정 보드에서 스크롤 시 브라우저가 정지되는 현상을 근본적으로 해결했습니다.

*   **현상 및 원인**:
    *   화면 진입 및 스크롤 시 수많은 `onScroll` 이벤트가 연달아 발생하여 가상화 영역 상태(`visibleColRange`)를 연속적으로 업데이트(SetState)함.
    *   이로 인해 거대한 DOM 매트릭스가 1초에 수십 번 반복 렌더링되면서 메인 스레드가 정지(Freeze)되는 병목 유발.
*   **최적화 기법 적용 (1안)**:
    *   **스크롤 디바운싱(Debouncing)**: 스크롤 진행 중에는 슬라이더바만 DOM 직접 조작으로 초고속 동기화하고, 무거운 가상화 영역 계산 및 렌더링 업데이트는 스크롤이 완전히 멈춘 후 80ms 뒤에 **단 1회**만 수행되도록 억제했습니다.
    *   **자동 스크롤 락(Auto Scroll Lock)**: `Today` 버튼 클릭 등을 통해 자동으로 스무스 스크롤이 이루어지는 동안에는 스크롤 이벤트 수신 및 상태 계산을 차단(`isAutoScrollingRef.current = true`)하고, 목적지 도달 시 해제하여 단 한 번의 화면 갱신만 일어나도록 방어 코드를 작성했습니다.

## 5. 테이블 행 가상화(Row Virtualization) 구현을 통한 성능 최적화
프로젝트 수와 소속 직원 수가 늘어남에 따라 생성되는 과도한 DOM 노드 개수로 인한 렌더링 성능 저하를 방지하기 위해 가상화 설계를 도입했습니다.

*   **Intersection Observer 기반 지연 렌더링 (Lazy Mount)**:
    *   화면에 현재 노출되지 않는 먼 거리에 위치한 프로젝트 행들의 리스트는 임시 뼈대 행(`<tr><td colSpan={150} style={{ height: estimatedHeight }} /></tr>`)으로 대체 렌더링하여 DOM 트리 크기를 약 90% 축소(18,000개 이상의 노드 -> 2,500개 수준)했습니다.
    *   이를 통해 브라우저의 스타일 계산, 레이아웃 정렬(Reflow/Repaint) 부하를 대폭 줄이고 대규모 스크롤 및 탭 전환 성능을 극대화했습니다.
    *   `rootMargin: '400px 0px'` 상하 버퍼 영역을 두어 사용자가 스크롤할 때 끊기거나 하얀 레이아웃 빈 공간이 보이지 않도록 부드러운 전환을 보장했습니다.
*   **프로젝트 뷰 & 그룹별 뷰 공통 적용**:
    *   **프로젝트별 뷰**: 개별 프로젝트 단위의 `ProjectItem`을 `LazyProjectRowWrapper`로 감싸 지연 로딩을 구현했습니다.
    *   **그룹별 뷰**: 테이블 표준을 만족하면서도 가상화를 실현하기 위해 기존의 단일 거대 `<tbody>` 구조에서, 그룹 헤더, 프로젝트 상세 리스트(`LazyProjectRowWrapper` 래핑), 그룹 요약 통계 행들을 각각 개별 `<tbody>` 블록으로 분할 연동하여 안전하고 모듈화된 격리 구조를 완성했습니다.

### 📂 변경 파일
*   `client/src/components/ProjectStatus.jsx`

### 🧪 검증 결과
*   Vite 빌드 컴파일 및 로컬 구동 브라우저 테스트 완료.
*   가상화 적용 후 DOM 노드가 급감하여 전체적인 렌더링 속도와 프레임이 눈에 띄게 부드러워졌으며, 프로젝트별/그룹별 화면 탭 전환 및 검색 필터링도 오작동 없이 실시간 갱신됨을 확인했습니다.

## 6. 그룹별 화면에서 인원 추가 시 상단 셀렉트된 부서(그룹) 기준 직원 필터링 구현
그룹별 배정 보드에서 인원을 추가할 때, 상단에 선택된 부서 또는 현재 렌더링 중인 부서 섹션 기준으로만 인원이 검색되도록 필터링을 최적화했습니다.

*   **부서 기반 직원 리스트 필터링 (`groupFilter` 연동)**:
    *   `getFilteredEmployees` 유틸리티 함수에 `groupFilter` 매개변수를 추가하여, 해당 인자값이 주어지면 해당 부서(그룹) 소속 직원들만 반환하도록 비즈니스 로직을 보완했습니다.
    *   `InlineAddRow` 및 `InlineSearchInput` 컴포넌트가 `groupFilter` prop을 주입받아 사용하도록 데이터 바인딩 구조를 설계했습니다.
*   **상황별 필터링 분기**:
    *   **그룹별 뷰**: 각 부서 섹션 하위의 프로젝트에서 `+ 인원 추가` 인풋을 활성화할 때, 현재 대상 부서의 명칭(`group.name`)을 `groupFilter`로 자동 전달하여 해당 부서원들만 검색 창에 노출되도록 제한했습니다. (상단 부서 필터가 'ALL'이든 특정 부서이든 상관없이 해당 섹션의 기준에 자동으로 대응합니다.)
    *   **프로젝트 뷰**: 기존의 전사 대상 인원 배정 시나리오를 그대로 유지하기 위해 `groupFilter`를 지정하지 않고 전체 직원을 대상으로 검색할 수 있도록 보존했습니다.

## 7. 인원 추가 시 새로 배정된 직원의 이름 셀로 포커스 자동 이동 구현
*   **현상 및 원인**: 기존에 프로젝트 배정 보드에서 직원을 새로 추가하면 포커스가 시작일 필드로 고정되어 있었으며, 이름 셀(`div`)에는 `tabIndex`가 없어 포커스가 가지 않는 현상이 있었습니다. 또한 그룹별 화면 렌더링 시 Ref 바인딩 누락 등으로 포커스 포지션이 엇갈렸습니다.
*   **해결 및 조치**:
    *   인원 배정 함수(`handleInlineAssign`)에서 신규 배정 멤버 생성 시 `setCursor`를 통해 `weekIndex: 'swap'`(이름 셀 컬럼)을 설정하도록 포커스 제어 타겟을 조정했습니다.
    *   `MemberRow` 및 `GroupMemberRow`의 이름 셀에 `tabIndex={0}` 및 전역 `cellRefs` 연결을 완료하여 포커스 접근성을 확보하고, 포커싱 시 파란 아웃라인을 주는 CSS를 적용했습니다.
    *   이름 셀 포커싱 상태에서 엔터(Enter) 입력 시 직원 변경 팝업이 활성화되도록 이벤트 리스너를 구현했습니다.

## 8. 엑셀 추출용 기간 입력 필드에 맞춤형 캘린더 피커(DatePicker) 적용
엑셀 추출 버튼 좌측의 날짜 입력 필드를 브라우저 기본 `<input type="date">`에서 기획에 맞춘 어두운 테마의 캘린더 피커 컴포넌트로 리팩토링했습니다.

*   **맞춤형 달력 드롭다운 개발**:
    *   상단 툴바의 주차 이동 달력 UI와 동일한 톤앤매너(어두운 배경, 연도/월 내비게이션, 깔끔한 폰트)를 가지며, 개별 일자 단위로 클릭하여 선택 가능한 달력 드롭다운을 React Portal 방식으로 새롭게 개발했습니다.
    *   **요일 색상 반영**: 토요일은 파란색, 일요일은 빨간색으로 표기하여 가시성을 높였습니다.
    *   **날짜 선택 및 호버 효과**: 현재 선택된 시작일/종료일 날짜에 둥근 사각형 보더 및 하이라이트 배경색을 씌우고 마우스 호버 애니메이션을 더해 프리미엄 UX를 완성했습니다.
*   **팝업 제어 및 예쁜 디스플레이**:
    *   기존 네이티브 인풋 창을 제거하고 캘린더 아이콘과 현재 날짜가 조합된 둥근 캡슐 버튼(`premium-date-input-display`)을 배치했습니다.
    *   달력 영역 또는 버튼 이외의 바깥 영역을 클릭할 때 달력이 자연스럽게 닫히도록 마우스 다운 이벤트 감지 훅(`useEffect`)을 보완했습니다.

## 9. 비즈니스 로직 커스텀 훅(useProjectStatus) 최종 이관 및 엑셀 다운로드(handleDownloadExcel) 기능 복원
*   **리팩토링 및 이관 완료**:
    *   `ProjectStatus.jsx` 컴포넌트 내부의 거대 비즈니스 로직 및 상태 제어 함수들을 `useProjectStatus.js` 커스텀 훅으로 최종 이관하여 관심사 분리(Separation of Concerns)를 성공적으로 마무리했습니다.
    *   Vite 빌드 시 발생하던 `isToolbarCollapsed` 미정의 ReferenceError 및 `COL_BUFFER` 초기화 TDZ(Temporal Dead Zone) 에러를 로컬 범위 재조정 및 순서 수정을 통해 완벽하게 해결했습니다.

## 10. useProjectStatus 훅 호출 구조분해 할당 시 user 변수 누락으로 인한 렌더링 에러 해결
*   **원인**: `useProjectStatus` 훅 내부에서는 로그인 정보(`user`)를 사용하고 리턴 값으로 `user`를 정상적으로 내보내고 있었으나, `ProjectStatus.jsx` 컴포넌트 쪽에서 `useProjectStatus()`를 호출할 때 구조분해 할당(Destructuring) 목록에 `user`가 누락되어 있었습니다. 이로 인해 컴포넌트 렌더링 도중 `user` 변수를 참조하는 코드가 실행될 때 `ReferenceError: user is not defined`가 발생해 화면이 뻗는 현상이 유발되었습니다.
*   **해결 및 조치**:
    *   `ProjectStatus.jsx` 1240행 부근의 구조분해 할당 목록 최상단에 `user` 변수를 안전하게 추가하여 컴포넌트 스코프 내로 올바르게 바인딩시켰습니다.
    *   에러 수정 후, 대시보드가 정상 로딩되며 모든 기능이 런타임 오류 없이 완전히 작동함을 최종 확인했습니다.

## 11. 영업현황(/sales) 화면 렌더링 성능 최적화 및 타이핑 렉(Lag) 개선
영업현황 시트에서 텍스트 입력 시 한 글자 단위로 발생하는 화면 프리징 현상을 해결하고, 데이터가 많아질 경우를 대비해 테이블 렌더링 부하를 획기적으로 개선했습니다.

*   **SpreadsheetCellInput 내부의 selection 갱신 useEffect 개선**:
    *   기존에 의존성 배열이 없이 선언되어 모든 렌더링 단계(타이핑 시점마다)마다 브라우저 selection 범위 연산을 강제로 무겁게 재실행하던 `useEffect`에 `[isFocused]` 의존성을 추가하여, 포커스가 진입하는 시점에만 선택 범위가 설정되도록 수정하여 타이핑 렉을 원천 제거했습니다.
*   **React.memo Props 동치성 보장을 위한 콜백 바인딩 최적화**:
    *   기존에는 `SalesDataRow` 렌더링 시 `onCommit`과 `onFocus` 콜백을 인라인 익명 화살표 함수(`onCommit={(v) => onCellChange(item.id, col.key, v)}`)로 내려주어 셀 컴포넌트의 `React.memo`가 무용지물화되어 렌더링이 대량 발생하던 구조를 개선했습니다.
    *   `SpreadsheetCellInput` 및 `SpreadsheetCellSelect`에 `rowId`와 `columnKey`를 직접 Props로 전달받도록 보강하여, 부모가 제공한 메모이즈된 단일 인스턴스 콜백(`onCommit={onCellChange}`)을 직접 바인딩할 수 있도록 수정했습니다.
    *   `handleCellFocus` 콜백을 `useCallback`으로 메모이즈 정의하여 추가하고 `onFocus` 핸들러에도 익명 함수를 걷어냈습니다.
*   **Intersection Observer 기반 영업현황 테이블 가상화(Row Virtualization) 구현**:
    *   각 영업 행들을 `LazySalesRowWrapper`로 감싸 지연 로딩을 구현했습니다.
    *   사용자 화면 가시 영역을 실시간 모니터링하여 화면 밖의 먼 영역에 위치한 행들을 빈 뼈대 형태의 `<tr>`로 돔에서 즉시 해제함으로써 다량의 데이터 렌더링 시에도 즉각적인 스크롤 반응 속도와 비약적으로 빠른 컴포넌트 마운트 스피드를 이끌어냈습니다.
